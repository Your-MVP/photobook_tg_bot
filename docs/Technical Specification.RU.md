**Техническое задание на разработку Telegram-бота «Photobook Bot» (MVP v0.1.0)**

**1. Цель проекта**
Разработать Telegram-бота, который позволяет пользователям собирать фотографии из личного чата с ботом или из семейного группового чата (куда пользователь самостоятельно добавляет бота), формировать фотокниги в формате PDF и подготавливать их для печати в типографии. Бот должен поддерживать управление списком фотокниг пользователя, хранение данных в базе и мониторинг администраторами через выделенную супергруппу.

**2. Основные функциональные требования**
- Приём фотографий: бот получает фото как в приватном чате, так и в групповых чатах (семейные альбомы), куда его добавили.
- Управление фотокнигами: команды /build (создание простой PDF-книги «одно фото — одна страница»), /clear, просмотр и управление списком книг пользователя.
- Хранение данных: PostgreSQL (SQLAlchemy async) — пользователи, сессии, список фото, готовые книги.
- Мониторинг через супергруппу: для каждого активного пользователя бот автоматически создаёт отдельную тему (forum topic) в заранее подготовленной супергруппе. В супергруппе находятся только администраторы (владельцы и назначенные модераторы), конечные пользователи туда не добавляются.

**3. Работа с супергруппой и темами (Telegram Bot API)**
3.1. Этапы создания супергруппы (выполняются только вручную):
- Владельцу бота необходимо вручную создать обычную группу и преобразовать её в супергруппу (chat type = supergroup).
- В настройках супергруппы включить режим «Темы» (forum = true).
- Добавить бота как администратора с правами: can_manage_topics, can_post_messages, can_edit_messages, can_delete_messages.
- Назначить дополнительных администраторов (людей) вручную через интерфейс Telegram.
- Получить chat_id супергруппы (отрицательное число) и message_thread_id главной темы.
- Сохранить chat_id супергруппы в .env-файл (SUPERGROUP_CHAT_ID) или в конфигурацию бота. Это единственный способ сделать супергруппу «известной» боту — Telegram Bot API не позволяет боту самостоятельно создавать или находить супергруппы.

3.2. Автоматические действия бота (после ручной настройки):
- При первом взаимодействии нового пользователя (/start) бот вызывает метод createForumTopic с параметрами: chat_id = SUPERGROUP_CHAT_ID, name = «User {user_id} — {username}», icon_color.
- Все последующие сообщения пользователя и действия бота (по желанию) форвардятся или дублируются в созданную тему с помощью message_thread_id.
- Администраторы видят список тем и могут отслеживать этапы работы с каждым пользователем.

**4. Приветствия и дифференциация пользователей**
- Стандартное приветствие (для обычных пользователей): короткое, дружелюбное сообщение с инструкцией по добавлению фото.
- Приветствие владельцу (bot owner, проверяется по user.id из config): расширенное, с доступом к административным командам и статистике.
- Приветствие назначенным администраторам супергруппы (проверяется по списку admin_ids в config): специальное сообщение с ссылками на все активные темы и статусами пользователей.

**5. Понимание возможностей и ограничений Telegram Bot API**
- Бот оперирует структурами: Chat, Message, MessageEntity, ForumTopic, User.
- Может: получать обновления через polling или webhook, отправлять фото/документы (PDF), создавать темы, управлять правами в темах.
- Не может: самостоятельно создавать супергруппы, находить существующие чаты без приглашения, читать историю чатов до добавления бота.
- Всё, что касается создания и первичной настройки супергруппы + добавления администраторов — только вручную. Автоматизировано только создание тем и работа внутри них.

**6. Дополнительные требования**
- Будущие этапы (не в MVP v0.1.0): подключение нейронной сети для отбора удачных фото и создания коллажей.
- Полная контейнеризация: Docker Compose (bot + Redis + PostgreSQL).
- Готовность к деплою на Oracle Cloud Always Free Tier (Arm Ampere A1).

**7. Требования к коду**
- aiogram 3.x, Redis FSM storage, async SQLAlchemy.
- Все строки интерфейса — только на английском (в соответствии с внутренним стандартом проекта).
- PEP 8, type hints, Google-style docstrings.
